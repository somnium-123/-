## 一、我的角色与职责

> *以下为各角色的参考模板，每人选择对应的填写*

### 刘子涵 — 项目经理（PM）

我在本项目中担任项目经理，核心职责包括：
1. **需求分析与拆解**：组织小组讨论，使用AI工具（ChatGPT/Copilot）辅助需求分析，编写《需求说明.md》，明确项目范围和功能边界。
2. **进度计划制定**：制定8天详细进度计划表，明确每天里程碑、交付物和汇报节点，每天跟踪各组员进度。
3. **Git仓库管理**：创建GitHub仓库，添加4名成员，配置main分支保护，制定Commit Message规范，确保每人每日至少1次有效提交。
4. **每日汇报组织**：组织4次正式汇报（Day1/2/4/6）和每日站立会，协调组员间的工作衔接和问题解决。
5. **测试与Bug管理**：制定25项测试用例清单，收集和分配Bug修复任务，跟踪修复进度，确保系统可运行。
6. **报告统筹**：整合各章节内容为完整的小组课程设计报告（≥5000字），统一格式和风格。

### 黄陈熙 — 领域建模师（DDD）

我在本项目中担任领域建模师，核心职责包括：
1. **领域分析**：学习DDD核心概念，分析"企业客户与合同管理"业务领域，识别核心业务概念和通用语言。
2. **限界上下文划分**：将系统划分为客户管理、联系人管理、合同管理三个限界上下文。
3. **实体与聚合设计**：设计Customer（聚合根）、Contact（实体）、Contract（聚合根）三个核心实体，定义属性、数据类型和关系。
4. **值对象识别**：识别Address（地址）、Money（金额）、ContactInfo（联系方式）三个值对象。
5. **ER图绘制**：使用Python matplotlib绘制高清ER实体关系图（含完整字段、外键标注、乌鸦脚符号、删除规则）。
6. **验证规则设计**：为全部实体属性配置Data Annotation验证特性，确保数据完整性。
7. **文档编写**：编写《领域模型说明.md》，包含完整的DDD设计说明、实体属性表、聚合设计、数据表映射和Seed数据。

### 关张旭 — 前端开发（Frontend）

我在本项目中担任前端开发，核心职责包括：
1. **共享布局设计**：设计并开发_Layout.cshtml共享布局页，包含Bootstrap 5导航栏、侧边栏（240px）、主内容区和页脚。
2. **Razor页面开发**：开发全部13个Razor页面（.cshtml），涵盖客户管理5个页面、联系人管理3个页面、合同管理5个页面、系统仪表盘首页。
3. **Bootstrap响应式**：实现三档响应式布局（桌面侧边栏/平板折叠/手机单栏），统一表格样式（striped+hover+responsive）、按钮风格、Badge标签。
4. **下拉框联动**：实现合同页面客户→联系人AJAX级联联动（jQuery监听change事件→GET JSON→动态渲染）。
5. **前端验证**：配置jQuery Validation Unobtrusive，为所有表单添加必填红色星号标识，实现即时字段验证反馈。
6. **操作提示**：添加操作成功（绿色Alert）和失败（红色Alert）的Bootstrap提示组件。
7. **系统截图**：截取全部功能页面截图，用于小组报告。

### 杨焱熙 — 后端开发（Backend）

我在本项目中担任后端开发，核心职责包括：
1. **Code First实体编码**：编写Customer.cs、Contact.cs、Contract.cs三个实体类，配置完整的Data Annotation验证特性。
2. **自定义验证**：编写DateAfterAttribute.cs自定义验证特性，确保回款日期不早于签订日期。
3. **DbContext配置**：创建AppDbContext.cs，使用Fluent API配置表名、索引、外键关系和三档删除规则（CASCADE/RESTRICT/SET NULL）。
4. **Migration管理**：执行Add-Migration InitSchema创建数据库迁移，执行Update-Database在SQL Server LocalDB中创建数据表。
5. **PageModel开发**：编写全部13个PageModel（.cshtml.cs），实现客户、联系人、合同三大模块的完整CRUD操作。
6. **联动API**：实现GetContactsByCustomer API接口，接收客户ID返回JSON格式的联系人列表。
7. **基础设施**：编写Program.cs（DI配置+Migration自动应用+Seed数据初始化）、appsettings.json（连接字符串）、SeedData.cs（4客户+6联系人+4合同示例数据）。
8. **Bug修复**：修复合同保存空引用、Seed数据重复插入、验证消息不统一等5个Bug。

### 张雨宣 — 后端助理（Backend Assist）

我在本项目中担任后端助理，核心职责包括：
1. **协助实体编码**：配合杨焱熙完成Contact.cs、DateAfterAttribute.cs等实体和验证特性的编码。
2. **Seed数据编写**：编写SeedData.cs中的示例客户（4个）、联系人（6个）和合同（4个）数据，涵盖不同行业、级别和状态。
3. **数据字典**：独立编写《数据字典.md》，完整记录Customers、Contacts、Contracts三张表的字段名、数据类型、约束、默认值和说明。
4. **API测试**：编写并执行合同模块联动API的测试用例，验证GetContactsByCustomer接口的正确性和性能。
5. **Bug发现与修复**：在集成测试中发现合同保存空引用Bug（P0），协助杨焱熙定位和修复。
6. **联调支持**：在Day 4合同模块开发中协助前后端进行联动API的接口联调和数据验证。
7. **文档整理**：整理API接口文档和验证规则汇总，补充后端技术说明。
