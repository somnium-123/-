# 企业客户与合同管理系统（CRM Lite）— 领域模型说明

> **编写人**：黄陈熙（领域建模师）  
> **编写日期**：2026年6月  
> **版本**：v1.0  
> **架构思想**：DDD（领域驱动设计）  

---

## 一、DDD 概述

本系统遵循**领域驱动设计（Domain-Driven Design, DDD）**思想进行建模。DDD 的核心是将业务领域中的概念直接映射为软件模型，通过**限界上下文（Bounded Context）**划分业务边界，通过**实体（Entity）**、**值对象（Value Object）**和**聚合（Aggregate）**构建领域模型。

---

## 二、限界上下文划分

经过对"企业客户与合同管理"业务领域的分析，识别出三个核心限界上下文：

```
┌─────────────────────────────────────────────────┐
│              企业客户与合同管理系统                │
│                                                  │
│  ┌──────────────┐  ┌──────────────┐  ┌────────┐ │
│  │ 客户管理上下文 │  │联系人管理上下文│  │合同管理│ │
│  │              │  │              │  │  上下文 │ │
│  │ Customer     │  │ Contact      │  │Contract│ │
│  │ Aggregate    │←─│ Entity       │←─│Aggregat│ │
│  │              │  │              │  │        │ │
│  └──────────────┘  └──────────────┘  └────────┘ │
│        ↑                ↑                ↑       │
│        │    关联关系     │    关联关系     │       │
│        └────────────────┴────────────────┘       │
└─────────────────────────────────────────────────┘
```

| 限界上下文 | 英文名称 | 核心职责 | 聚合根 |
|-----------|----------|----------|--------|
| 客户管理上下文 | Customer Context | 管理客户基本信息、行业分类、客户级别等 | Customer |
| 联系人管理上下文 | Contact Context | 管理客户下的联系人信息，维护联系人与客户的关联 | Contact（属于Customer聚合） |
| 合同管理上下文 | Contract Context | 管理合同的签订、执行和终止，跟踪合同金额与回款计划 | Contract |

## 三、实体设计（Entity）

实体是拥有**唯一标识符**的领域对象，其标识在其整个生命周期中保持不变。

### 3.1 Customer（客户）— 聚合根

```
┌─────────────────────────────────────┐
│           Customer                   │
│         <<Aggregate Root>>           │
├─────────────────────────────────────┤
│ + CustomerId : int          (PK)    │
│ + Code : string             (唯一)   │
│ + Name : string             (必填)   │
│ + Industry : string                  │
│ + Scale : string                     │
│ + Level : CustomerLevel     (枚举)   │
│ + Region : string                    │
│ + Address : string                   │
│ + PostalCode : string                │
│ + Phone : string                     │
│ + Website : string                   │
│ + Remark : string                    │
│ + CreatedAt : DateTime               │
├─────────────────────────────────────┤
│ + Contacts : List<Contact>  (导航)   │
│ + Contracts : List<Contract>(导航)   │
└─────────────────────────────────────┘
```

**关键属性**：

| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| CustomerId | int | PK, Identity | 唯一标识，自增主键 |
| Code | string(20) | Required, Unique | 客户编号，格式：CUS-YYYYMMDD-XXX |
| Name | string(100) | Required | 客户企业全称 |
| Industry | string(50) | Nullable | 所属行业（制造业/服务业/IT/…） |
| Scale | string(20) | Nullable | 企业规模（小微企业/中型企业/大型企业） |
| Level | enum | Required | 客户级别：普通(0)/重要(1)/VIP(2) |
| Region | string(50) | Nullable | 所在地区（华东/华南/华北/…） |
| Address | string(200) | Nullable | 详细地址 |
| PostalCode | string(10) | Nullable | 邮政编码 |
| Phone | string(20) | Nullable | 企业联系电话 |
| Website | string(100) | Nullable | 企业官网网址 |
| Remark | string(500) | Nullable | 备注说明 |
| CreatedAt | DateTime | Required | 创建时间，默认当前时间 |

**CustomerLevel 枚举**：
```csharp
public enum CustomerLevel
{
    Normal = 0,    // 普通客户
    Important = 1, // 重要客户
    VIP = 2        // VIP客户
}
```

### 3.2 Contact（联系人）— 实体

```
┌─────────────────────────────────────┐
│           Contact                    │
│          <<Entity>>                  │
├─────────────────────────────────────┤
│ + ContactId : int          (PK)     │
│ + CustomerId : int         (FK)     │
│ + Name : string            (必填)   │
│ + Position : string                  │
│ + Department : string                │
│ + Phone : string                     │
│ + Email : string                     │
│ + IsPrimary : bool                   │
├─────────────────────────────────────┤
│ + Customer : Customer      (导航)   │
│ + Contracts : List<Contract>(导航)  │
└─────────────────────────────────────┘
```

**关键属性**：

| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| ContactId | int | PK, Identity | 唯一标识，自增主键 |
| CustomerId | int | FK → Customer, Required | 所属客户外键 |
| Name | string(50) | Required | 联系人姓名 |
| Position | string(50) | Nullable | 职务（如"采购经理"） |
| Department | string(50) | Nullable | 所属部门 |
| Phone | string(20) | Nullable | 联系电话 |
| Email | string(100) | Nullable | 电子邮箱 |
| IsPrimary | bool | Required, Default=false | 是否主要联系人 |

### 3.3 Contract（合同）— 聚合根

```
┌─────────────────────────────────────┐
│           Contract                   │
│       <<Aggregate Root>>             │
├─────────────────────────────────────┤
│ + ContractId : int         (PK)     │
│ + ContractCode : string    (唯一)   │
│ + Name : string            (必填)   │
│ + CustomerId : int         (FK)     │
│ + ContactId : int?         (FK可选) │
│ + Amount : decimal(18,2)   (必填)   │
│ + SignedDate : DateTime    (必填)   │
│ + ExpectedPaymentDate : DateTime?   │
│ + Status : ContractStatus  (枚举)   │
│ + Remark : string                    │
├─────────────────────────────────────┤
│ + Customer : Customer      (导航)   │
│ + Contact : Contact?       (导航)   │
└─────────────────────────────────────┘
```

**关键属性**：

| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| ContractId | int | PK, Identity | 唯一标识，自增主键 |
| ContractCode | string(20) | Required, Unique | 合同编号，格式：CON-YYYYMMDD-XXX |
| Name | string(200) | Required | 合同名称 |
| CustomerId | int | FK → Customer, Required | 关联客户（必选） |
| ContactId | int? | FK → Contact, Nullable | 关联联系人（可选） |
| Amount | decimal(18,2) | Required, Range(0.01, …) | 合同金额，必须>0 |
| SignedDate | DateTime | Required | 合同签订日期 |
| ExpectedPaymentDate | DateTime? | Nullable | 预计回款日期 |
| Status | enum | Required | 合同状态 |
| Remark | string(500) | Nullable | 附件/备注说明 |

**ContractStatus 枚举**：
```csharp
public enum ContractStatus
{
    InProgress = 0, // 执行中
    Completed = 1,  // 已完成
    Terminated = 2  // 已终止
}
```

## 四、值对象设计（Value Object）

值对象是**没有唯一标识**的领域对象，通过其属性值来定义相等性。值对象通常是**不可变的**。

### 4.1 Address（地址值对象）

```
┌──────────────────────────┐
│       Address             │
│    <<Value Object>>       │
├──────────────────────────┤
│ + Province : string       │
│ + City : string           │
│ + District : string       │
│ + Detail : string         │
│ + PostalCode : string     │
├──────────────────────────┤
│ + ToString() : string     │
└──────────────────────────┘
```

> **设计说明**：Address 值对象嵌入 Customer 聚合中使用。当前版本中，考虑到简化实现，地址相关字段直接作为 Customer 的属性。在后续迭代中可重构为值对象。

### 4.2 Money（金额值对象）

```
┌──────────────────────────┐
│        Money              │
│    <<Value Object>>       │
├──────────────────────────┤
│ + Amount : decimal        │
│ + Currency : string       │
├──────────────────────────┤
│ + ToString() : string     │
│ + Add(Money) : Money      │
└──────────────────────────┘
```

> **设计说明**：Money 值对象封装了金额与币种，用于 Contract 的合同金额字段。提供金额格式化（如 ¥50,000.00）和算术运算能力。当前版本默认币种为 CNY（人民币）。

### 4.3 ContactInfo（联系方式值对象）

```
┌──────────────────────────┐
│      ContactInfo          │
│    <<Value Object>>       │
├──────────────────────────┤
│ + Phone : string          │
│ + Email : string          │
├──────────────────────────┤
│ + IsValid() : bool        │
└──────────────────────────┘
```

> **设计说明**：ContactInfo 可复用于 Customer 和 Contact 实体。当前版本中，为简化实现，电话和邮箱字段直接作为实体属性。后续可提取为值对象。

## 五、聚合设计（Aggregate）

聚合是一组相关对象的集合，将数据修改的范围限定在聚合边界内，保证业务规则的一致性。**聚合根是聚合的唯一入口**，外部对象只能通过聚合根访问聚合内的实体。

### 5.1 Customer 聚合

```
┌──────────────────────────────────────┐
│         Customer 聚合                  │
│                                      │
│  ┌────────────────────┐              │
│  │  Customer          │              │
│  │  <<Aggregate Root>>│              │
│  └───────┬────────────┘              │
│          │ 1                          │
│          │ has                        │
│          │ N                          │
│  ┌───────┴────────────┐              │
│  │  Contact           │              │
│  │  <<Entity>>        │              │
│  └────────────────────┘              │
│                                      │
│  业务不变性约束：                      │
│  • 同一客户下不能有完全相同的联系人     │
│    （姓名 + 电话 相同视为重复）        │
│  • 客户删除时，关联联系人同步删除       │
│  • 客户有关联合同时，不允许删除         │
└──────────────────────────────────────┘
```

| 设计决策 | 说明 |
|----------|------|
| 为什么 Contact 在 Customer 聚合内？ | 联系人的生命周期依赖于客户——客户不存在了，联系人就没有存在的意义。Contact 通过 Customer 聚合根来访问和修改。 |
| Customer 删除规则 | CASCADE 删除 Contact；但如果有 Contract 存在，RESTRICT 删除 Customer |

### 5.2 Contract 聚合

```
┌──────────────────────────────────────┐
│         Contract 聚合                  │
│                                      │
│  ┌────────────────────┐              │
│  │  Contract          │              │
│  │  <<Aggregate Root>>│              │
│  └────────────────────┘              │
│                                      │
│  业务不变性约束：                      │
│  • 合同金额必须 > 0                   │
│  • 签订日期不能晚于预计回款日期        │
│  • 合同关联的客户必须存在于系统中      │
│  • 合同关联的联系人必须属于所选客户    │
└──────────────────────────────────────┘
```

### 5.3 聚合间关系

```
┌──────────────┐                   ┌──────────────┐
│   Customer   │────── 引用Id ────→│   Contract   │
│   聚合根      │←──── 引用Id ─────│   聚合根      │
└──────────────┘                   └──────────────┘
       │                                  │
       │ 直接持有                          │ 引用Id
       ▼                                  ▼
┌──────────────┐                   ┌──────────────┐
│   Contact    │←──── 引用Id ─────│  (Contract   │
│   实体        │                   │   .ContactId)│
└──────────────┘                   └──────────────┘
```

> **关键设计原则**：跨聚合引用使用**标识符（Id）**而非对象引用。Contract 通过 CustomerId 引用 Customer，通过 ContactId 引用 Contact。这确保了聚合间的松耦合。

## 九、Data Annotation 配置

### Customer 验证规则

```csharp
public class Customer
{
    [Key]
    public int CustomerId { get; set; }

    [Required(ErrorMessage = "客户编号不能为空")]
    [StringLength(20)]
    public string Code { get; set; }

    [Required(ErrorMessage = "客户名称不能为空")]
    [StringLength(100, ErrorMessage = "客户名称最长100个字符")]
    [Display(Name = "客户名称")]
    public string Name { get; set; }

    [StringLength(50)]
    [Display(Name = "所属行业")]
    public string Industry { get; set; }

    [StringLength(20)]
    [Display(Name = "企业规模")]
    public string Scale { get; set; }

    [Required]
    [Display(Name = "客户级别")]
    public CustomerLevel Level { get; set; } = CustomerLevel.Normal;

    [StringLength(50)]
    [Display(Name = "所在地区")]
    public string Region { get; set; }

    [StringLength(200)]
    [Display(Name = "详细地址")]
    public string Address { get; set; }

    [StringLength(10)]
    [Display(Name = "邮政编码")]
    public string PostalCode { get; set; }

    [Phone(ErrorMessage = "联系电话格式不正确")]
    [StringLength(20)]
    [Display(Name = "联系电话")]
    public string Phone { get; set; }

    [Url(ErrorMessage = "网址格式不正确")]
    [StringLength(100)]
    [Display(Name = "企业网址")]
    public string Website { get; set; }

    [StringLength(500)]
    [Display(Name = "备注说明")]
    public string Remark { get; set; }

    [Required]
    [Display(Name = "创建时间")]
    public DateTime CreatedAt { get; set; } = DateTime.Now;

    // 导航属性
    public List<Contact> Contacts { get; set; }
    public List<Contract> Contracts { get; set; }
}
```

### Contract 验证规则

```csharp
public class Contract
{
    [Key]
    public int ContractId { get; set; }

    [Required]
    [StringLength(20)]
    public string ContractCode { get; set; }

    [Required(ErrorMessage = "合同名称不能为空")]
    [StringLength(200, ErrorMessage = "合同名称最长200个字符")]
    [Display(Name = "合同名称")]
    public string Name { get; set; }

    [Required(ErrorMessage = "请选择客户")]
    [Display(Name = "关联客户")]
    public int CustomerId { get; set; }

    [Display(Name = "关联联系人")]
    public int? ContactId { get; set; }

    [Required(ErrorMessage = "合同金额不能为空")]
    [Range(0.01, 99999999.99, ErrorMessage = "合同金额必须在0.01~99999999.99之间")]
    [Display(Name = "合同金额")]
    [Column(TypeName = "decimal(18,2)")]
    public decimal Amount { get; set; }

    [Required(ErrorMessage = "签订日期不能为空")]
    [Display(Name = "签订日期")]
    [DataType(DataType.Date)]
    public DateTime SignedDate { get; set; }

    [Display(Name = "预计回款日期")]
    [DataType(DataType.Date)]
    public DateTime? ExpectedPaymentDate { get; set; }

    [Required]
    [Display(Name = "合同状态")]
    public ContractStatus Status { get; set; } = ContractStatus.InProgress;

    [StringLength(500)]
    [Display(Name = "备注说明")]
    public string Remark { get; set; }

    // 导航属性
    public Customer Customer { get; set; }
    public Contact Contact { get; set; }
}
```
