## 一、项目背景

某制造企业在日常运营中面临客户信息管理分散、联系人记录不统一、合同台账依赖Excel手工维护等突出问题。随着企业业务规模的扩大，传统的Excel管理方式已无法满足数据一致性、查询效率和协作管理的需求。企业急需一套轻量级的客户关系管理系统（CRM），将客户信息、联系人、合同及回款计划进行统一管理，替代原有Excel台账，提高数据的准确性和查询效率。

本系统定位为企业内部业务运营系统（Line of Business, LOB），聚焦于核心业务数据的管理，不涉及复杂的权限控制与在线支付功能。系统采用ASP.NET Core 7框架开发，遵循领域驱动设计（DDD）思想，以Code First模式构建数据访问层，确保代码与数据库的同步演进。

## 二、需求分析

### 2.1 功能需求

系统核心功能包括四大模块：

**客户管理模块**：支持客户信息的增删改查操作，包含客户名称、编号（自动生成）、所属行业、企业规模、客户级别（普通/重要/VIP）、所在地区、详细地址、联系电话、企业网址、备注说明等13个属性字段。客户列表支持分页显示（每页10条），提供按客户名称和行业的搜索功能。客户删除时需检查关联合同，存在合同时阻止删除。

**联系人管理模块**：支持为客户添加多个联系人，联系人信息包括姓名、职务、所属部门、联系电话、电子邮箱、是否主要联系人等8个属性字段。联系人列表支持按客户进行筛选展示。联系人通过外键CustomerId关联到客户，删除客户时联系人同步级联删除。

**合同管理模块**：支持合同的创建、编辑、删除和详情查看，合同信息包括合同编号（自动生成）、合同名称、关联客户、关联联系人（可选）、合同金额、签订日期、预计回款日期、合同状态（执行中/已完成/已终止）、备注说明等10个属性字段。合同金额以人民币格式（¥#,##0.00）显示，合同状态以不同颜色的Badge标签展示。核心亮点功能是客户-联系人下拉框联动——选择客户后系统通过AJAX动态加载该客户下的联系人列表。

**表单验证模块**：采用前端jQuery Validation Unobtrusive和后端Data Annotation双重验证机制。所有必填字段（如客户名称、合同金额）均有[Required]特性标注，特殊格式字段（如Phone、Email、Url）均有对应验证特性。前端必填字段以红色星号标识，验证失败时字段下方显示红色错误提示，表单顶部显示验证摘要。

### 2.2 非功能需求

系统需满足以下非功能性指标：页面响应时间在正常负载下不超过3秒；界面简洁直观，符合企业级后台管理系统的操作习惯；支持Chrome、Edge等主流浏览器的最新版本；代码遵循DDD分层架构，模块间低耦合，便于后续功能扩展；通过EF Core的Fluent API和外键约束保证数据参照完整性；前端采用Bootstrap 5响应式框架，适配桌面、平板和手机三种屏幕尺寸。

## 三、DDD领域驱动设计

### 3.1 限界上下文划分

依据DDD战略设计思想，本系统划分为三个限界上下文：

**客户管理上下文（Customer Context）**：管理客户基本信息及其行业分类，维护客户的生命周期状态。核心聚合为Customer（客户聚合根）。

**联系人管理上下文（Contact Context）**：管理客户下的联系人信息，维护联系人与客户的关联关系。Contact实体属于Customer聚合内部，通过Customer聚合根访问。

**合同管理上下文（Contract Context）**：管理合同的签订、执行和终止，跟踪合同金额与回款计划。核心聚合为Contract（合同聚合根），通过标识符引用Customer和Contact。

### 3.2 实体与聚合设计

**Customer（客户聚合根）**：拥有全局唯一标识CustomerId，包含13个属性。Customer作为聚合根管理其内部的Contact集合。关键业务不变性约束包括：同一客户下不能存在姓名+电话完全相同的联系人；客户删除时若存在关联合同则阻止删除。

**Contact（联系人实体）**：属于Customer聚合内部实体，拥有唯一标识ContactId，通过CustomerId外键关联客户。一个客户可拥有多个联系人。

**Contract（合同聚合根）**：拥有全局唯一标识ContractId，通过CustomerId和ContactId（可选）外键引用客户和联系人。关键业务不变性约束：合同金额必须大于0；签订日期不能晚于预计回款日期；合同关联的客户必须存在于系统中。

### 3.3 值对象

识别出三个值对象：Address（地址，由省/市/区/详情/邮编组成）、Money（金额，由数值和币种组成）、ContactInfo（联系方式，由电话和邮箱组成）。当前版本为简化实现，值对象字段直接作为实体属性，后续可重构提取。

## 四、Code First实现

### 4.1 实体编码

严格按照Code First模式进行数据库开发。首先设计领域实体（C# Class），包含完整的Data Annotation验证特性。实体类定义在Models文件夹中，包括Customer.cs、Contact.cs、Contract.cs以及DateAfterAttribute.cs自定义验证特性。

Customer实体配置了[Key]、[Required]、[StringLength]、[Phone]、[Url]等验证特性。Contact实体配置了外键关联和[EmailAddress]验证。Contract实体配置了[Range]金额范围和自定义[DateAfter]日期验证。

### 4.2 DbContext与Fluent API

AppDbContext类继承自DbContext，通过DbSet<T>属性暴露三个实体集。在OnModelCreating方法中使用Fluent API进行完整的实体映射配置，包括：表名映射（ToTable）、主键配置（HasKey）、唯一索引（HasIndex.IsUnique）、外键关系与删除行为（HasOne...WithMany...HasForeignKey...OnDelete）。

三个核心外键删除规则：Contact.CustomerId → ON DELETE CASCADE（删除客户时级联删除联系人）；Contract.CustomerId → ON DELETE RESTRICT（有关联合同时禁止删除客户）；Contract.ContactId → ON DELETE SET NULL（删除联系人时合同引用自动置空）。

### 4.3 Migration与Seed数据

执行Add-Migration InitSchema命令创建初始迁移文件，执行Update-Database将迁移应用到SQL Server LocalDB数据库。SeedData类在应用启动时自动检查并初始化4个示例客户、6个示例联系人和4个示例合同，涵盖不同行业、级别和合同状态，便于系统演示。

## 五、前端页面设计

### 5.1 整体布局

系统采用标准后台管理布局，包含固定顶部导航栏（显示系统名称和日期）、左侧侧边栏导航（240px宽，含客户管理/联系人管理/合同管理三个菜单项，当前页高亮）、主内容区（Bootstrap container）、底部页脚（版权和团队信息）。

### 5.2 UI框架

使用Bootstrap 5.3作为CSS框架，Bootstrap Icons作为图标库。配色方案采用Bootstrap默认主题色：主色蓝色（#0d6efd）、成功绿色（#198754）、警告黄色（#ffc107）、危险红色（#dc3545）。表格使用striped条纹样式和hover悬停高亮。按钮统一使用Bootstrap标准按钮样式。客户级别以Badge标签区分（VIP=红色、重要=黄色、普通=灰色），合同状态以Badge标签区分（执行中=蓝色、已完成=绿色、已终止=灰色）。

### 5.3 响应式设计

通过Bootstrap栅格系统实现三档响应式布局：桌面端（≥992px）侧边栏+主内容区双栏布局；平板端（768-991px）侧边栏折叠为汉堡菜单；手机端（<768px）单栏布局，表格启用横向滚动（table-responsive）。

### 5.4 下拉框联动（核心亮点）

合同新增/编辑页面实现了客户到联系人的AJAX级联下拉框联动。用户选择客户后，前端通过jQuery监听change事件，发起GET请求到Contracts/Create?handler=ContactsByCustomer&customerId=xxx，后端返回该客户关联的联系人JSON列表，前端动态填充联系人下拉框。这一交互显著提升了数据录入的准确性和用户体验。

## 六、测试与验证

### 6.1 测试方案

系统集成测试共设计25个测试用例，覆盖五大测试维度：客户管理模块（11项）、联系人管理模块（7项）、合同管理模块（9项）、数据完整性（5项）、UI/响应式（5项）。测试采用手工测试方式，逐功能模块执行测试用例，记录测试结果。数据验证通过直接查询SQL Server数据库确认数据正确写入。

### 6.2 测试结果

25项测试用例全部通过，测试通过率100%。发现并修复5个Bug：合同保存时空引用异常（P0）、客户删除页未显示关联合同数（P1）、移动端表格溢出（P2）、Seed数据重复插入（P0）、验证错误消息不统一（P2）。所有Bug均在集成测试阶段修复并回归验证通过。

## 八、项目总结

### 8.1 主要成果

经过8天集中实训，小组成功完成了企业客户与合同管理系统（CRM Lite）的设计与开发。系统实现了客户管理、联系人管理、合同管理三大核心模块的完整CRUD功能，包含25项测试用例全部通过的可运行系统。项目严格遵循Code First + Migration开发规范，采用DDD领域驱动设计思想进行架构设计，前端实现了Bootstrap 5响应式布局和客户-联系人下拉框联动等亮点功能。

项目累计产生44个代码文件（含13个C# PageModel、13个Razor页面、4个实体类、3个基础设施文件等）和7个文档文件（需求说明、进度计划、领域模型说明、ER图、数据字典、测试用例等）。Git仓库共有15次有效提交，5名成员各有独立贡献。

### 8.2 技术收获

通过本课程设计，小组成员在以下方面获得了显著提升：掌握了ASP.NET Core 7 + EF Core的Code First开发流程；理解了DDD领域驱动设计的核心概念（限界上下文、实体、值对象、聚合）；熟练运用了Data Annotation和Fluent API进行数据模型配置；掌握了Razor Pages页面开发模式和Bootstrap 5响应式布局；学会了使用Git进行团队协作和版本管理；理解了前后端数据验证的协同机制和AJAX异步交互。

### 8.3 不足与改进

项目存在以下可改进之处：当前未实现用户登录和权限管理功能；联系人删除功能需通过Controller而非直接页面操作；部分值对象（Address、Money）尚未从实体中独立提取；未编写自动化单元测试。这些将在后续学习和项目中进一步完善。
